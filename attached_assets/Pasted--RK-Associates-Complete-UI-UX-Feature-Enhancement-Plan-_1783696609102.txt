## 📱 2. Mobile-First UI/UX Overhaul (HCI Principles)

### Problem: Everything is too large & overly scrollable

**Solutions:**

| Issue | Fix |
|-------|-----|
| **Oversized elements** | Reduce padding, font sizes, card gaps on mobile using Tailwind `sm:` `md:` breakpoints |
| **Too much scrolling** | Implement **sticky filters** (collapsible), **infinite scroll** or **pagination** (10 per page) |
| **Huge filter bar** | Collapse filters into a **slide-out drawer** (icon-based) — opens on click, doesn't take screen space |
| **Cluttered cards** | Reduce card height — show only: image, title, price, badge, location. Expand on tap for details |
| **Navigation** | Fixed **bottom navigation bar** (mobile) with 5 tabs: Home, Properties, Near By, Favorites, WhatsApp |

**HCI Principles Applied:**
- **Fitts's Law:** Buttons are appropriately sized for thumbs (min 44px touch targets)
- **Hick's Law:** Fewer visible options, more hidden behind icons
- **Consistency:** Same UI patterns across all pages
- **Feedback:** Haptic/tactile feedback on button taps

## ❤️ 4. Save to Favorites (No Login Required)

**Implementation:**

## 🎨 7. Dark Mode & Color Adjustments

**Dark Mode Fix:**
- All text: `text-slate` (light) | `dark:text-white` (dark)
- Headings: `text-navy` (light) | `dark:text-beige` (dark)
- Cards: `bg-white` (light) | `dark:bg-dark-card` (dark)
- Ensure contrast ratio ≥ 4.5:1 on all text

**Color Palette Consistency:**
- Buttons: Gold gradient for CTAs
- Badges: Sale = Gold, Rent = Navy, Both = Split
- Borders & shadows: Subtle, consistent
